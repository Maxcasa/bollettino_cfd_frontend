# Screenshot Guide

Save all documentation screenshots in this folder.

## Required file names

Use these exact names so the placeholders in the docs resolve automatically:

1. `01-login.png`
2. `02-base-home.png`
3. `03-meteo-select.png`
4. `04-meteo-editor-grid.png`
5. `05-cfd-select.png`
6. `06-cfd-editor-maps-a.png`
7. `06-cfd-editor-maps-b.png`
8. `06-cfd-editor-maps-c.png`

## What to capture

1. `01-login.png`: Login form with username/password fields visible.
2. `02-base-home.png`: Home area after login with top navigation bar visible.
3. `03-meteo-select.png`: Meteo bulletin selection table and create controls.
4. `04-meteo-editor-grid.png`: Meteo OGGI/DOMANI grid with some colored cells.
5. `05-cfd-select.png`: CFD page with meteo list and CFD list side by side.
6. `06-cfd-editor-maps-a.png`: CFD editor map view for risk tab A.
7. `06-cfd-editor-maps-b.png`: CFD editor map view for risk tab B.
8. `06-cfd-editor-maps-c.png`: CFD editor map view for risk tab C.

## Capture quality recommendations

- Preferred resolution: 1920x1080 (or 1600x900 minimum).
- Use PNG format.
- Use browser zoom 100%.
- Crop only browser chrome if needed; keep key UI controls visible.
- Avoid personal data in screenshots.
- Keep language and theme consistent across all captures.

## Workflow

1. Take screenshots in the running application.
2. Rename each file to the required name.
3. Save files in this folder.
4. Commit and push.

After adding these files, image placeholders in the docs will render automatically on GitHub.
